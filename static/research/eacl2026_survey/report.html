ReciFine: Finely Annotated Recipe Dataset for Controllable Recipe Generation

Nuhu Ibrahim, Rishi Ravikumar, Robert Stevens, Riza Batista-Navarro
EACL 2026 Findings University of Manchester GitHub: ReciFine Food NLP
[Figure] See paper PDF — ReciFine pipeline overview (NER → large-scale annotation → controllable generation)

1. 전체 요약

본 논문은 레시피 도메인 최초의 대규모 fine-grained entity annotation 데이터셋을 구축한다. RecipeNLG 2.2M 레시피에 10가지 entity type(Food, Tool, Duration, Quantity, Action by chef/food/tool, Discontinuous action, Food state, Tool state)으로 9,700만+ entity를 자동 추출하고, 이를 활용하여 entity 조합에 따라 제어 가능한 레시피 생성을 수행한다. 핵심 발견: RecipeNLG의 92% 레시피에서 ingredient list에 있는 재료가 instruction에 누락되어 있어, instruction-level NER이 필수적임을 밝힌다. Fine-tuned Mistral 7B가 GPT-4.1 few-shot을 크게 능가한다.

2. 기존 연구 흐름

  • Recipe1M+ (Marin+, 2018): 수백만 레시피+이미지, entity annotation 없음
  • RecipeNLG (Bien+, 2020): 2.2M 레시피, food entity만 ingredient list에서 추출, instruction 무시
  • ERFG (Yamakata+, LREC 2020): 300 레시피, 10개 entity type 상세 annotation — ReciFine의 schema 원천
  • RecipeMC (Taneja+, 2024): GPT-2 + MCTS 레시피 생성
  • LLaVA-Chef (Mohbat & Zaki, 2024): multimodal 레시피 생성

3. 이 논문의 차별점

  • Scale + Granularity: 기존 fine-grained dataset(ERFG)은 300편, ReciFine은 2.2M편에 10 entity type 적용
  • Instruction-level NER: ingredient list가 아닌 조리 instruction에서 entity 추출 — 92% coverage gap 해결
  • 5가지 Entity-Enhanced Prompt Types: entity 조합별 제어 가능한 생성 (ING-List, ING-Inst, ING+Tool, ING+Cook, ALL)
  • Fine-tuned 7B > GPT-4.1 few-shot: 도메인 특화 fine-tuning의 우월성 실증

4. Method 상세

Stage A: Recipe NER 시스템 구축

  • ERFG 300편 + 수동 annotation 500편 = 800편 학습 데이터
  • Knowledge-Augmented (KA): entity type 정의/예시를 프롬프트에 prepend — [CLS] knowledge_context [SEP] recipe_text [SEP]
  • Entity Type-Specific (ETS): 타입별 독립 binary BIO 학습 → 유사 타입 간 혼동 감소
  • 최고 모델: RecipeRoBERTa-KA → ERFG test F1 = 90.37

Stage B: 대규모 적용

  • RecipeRoBERTa-KA로 RecipeNLG 2.2M 전체 자동 annotation → 9,700만+ entity (silver-standard)

Stage C: Controllable Generation

  • 5가지 EPT (Entity-Enhanced Prompt Type)별 레시피 생성
  • Mistral 7B + LoRA fine-tuning, 200K 학습 데이터

7. 결론

레시피 instruction-level의 fine-grained NER이 controllable generation의 핵심이며, entity type이 많을수록 자동 메트릭은 단조 증가한다. 다만 human evaluation과의 괴리가 존재하여, 모든 entity를 사용하는 것이 항상 최선은 아님을 시사한다. Fine-tuned 소형 모델이 대형 모델 few-shot을 일관되게 능가한다.

8. 내 의견

[Strengths]

  • 2.2M 규모의 10-type fine-grained 레시피 데이터셋은 커뮤니티에 큰 기여
  • 92% coverage gap 발견은 기존 연구의 근본 문제를 지적
  • Entity 조합별 controllable generation 프레임워크가 체계적

[Weaknesses]

  • 9,700만 entity는 silver-standard (모델 예측, F1=83~95) — 수백만 개 오류 가능
  • Entity 스키마는 ERFG에서 그대로 차용 — 새 entity type 제안 없음
  • Human eval vs auto eval 괴리: ING-Inst ≈ ALL in human eval, "more types = better" 논거 약화
  • KG 통합 없음: 저자도 future work로 언급
  • 평가 규모 작음: 2.2M 중 200K fine-tuning, 200개만 평가

[Relevance]

  • LG Oven 10개 entity type이 조리 절차 지식(도구, 시간, 동작, 상태)과 정확히 일치 — 오븐 Agent 직결
  • SoC RAG Fine-tuned 7B > GPT-4.1 few-shot — on-device 경량 모델 전략 근거
  • KG 기초연구 ReciFine entity → KG 구조화하면 후속 논문 가능 (저자가 못 한 gap)
  • K-Culture 한식 레시피에 동일 스키마 적용 가능 (NER 모델 transfer 필요)
  • Glocal Lab 다국어 레시피 NER 확장 가능성

Knowledge Augmentation Enhances Token Classification for Recipe Understanding

Nuhu Ibrahim, Robert Stevens, Riza Batista-Navarro
EACL 2026 Long University of Manchester GitHub: ReciFine Food NER
[Figure] See paper PDF — Knowledge-Augmented input format: [CLS] knowledge [SEP] recipe [SEP]

1. 전체 요약

ReciFine의 자매 논문으로, Knowledge-Augmented (KA) input과 Entity Type-Specific (ETS) 학습을 통한 레시피 NER 방법론을 제안한다. 7개 food NER 데이터셋 중 5개에서 SOTA를 달성하며, RecipeRoBERTa-KA가 GPT-4o 4-shot을 F1 28점 차이로 압도 (89.96 vs 61.76)한다. 150만+ annotation instance를 포함하는 통합 데이터셋을 공개한다.

2. 기존 연구 흐름

  • Banerjee+ (2021, ACM Trans): biomedical NER에 외부 knowledge context — 직접 영감
  • ERFG (Yamakata+, LREC 2020): 300 레시피, 10 entity type
  • TASTEset (Wroblewska+, 2022): 9 entity type, BERT baseline
  • Goel+ (2024): spaCy-transformer 기반 recipe NER
  • FoodIE (Popovski+, 2019): rule-based food entity extraction

3. 이 논문의 차별점

  • ETS 학습: multi-entity BIO → 타입별 binary BIO 분리, 유사 타입 간 혼동 제거
  • KA input: entity type 정의/예시/질문을 BERT segment-A에 prepend — 5가지 knowledge context 타입
  • 7개 corpus 통합: FINER, AllRecipes, GeniusKitchen, FoodBase, ERFG, TASTEset 1&2
  • BERT-base >> GPT-4o: domain-specific encoder가 general LLM을 대폭 능가

4. Method 상세

Entity Type-Specific Training

  • 기존: 하나의 모델이 BIO-|E| (모든 entity type) 동시 예측
  • 제안: entity type별 독립 forward pass, BIO 3-class만 예측
  • 장점: "action by chef" vs "action by food" 같은 유사 타입 혼동 방지
  • 단점: N개 entity type이면 inference N배 비용 (논문에서 논의 안 함)

7. 결론

Knowledge-augmented input과 entity type-specific training의 조합이 food NER에서 강력한 성능을 보인다. 도메인 특화 encoder 모델(BERT-base급)이 GPT-4o를 28점 차이로 압도하며, 이는 on-device 배포 시나리오에서 중요한 시사점을 제공한다.

8. 내 의견

[Strengths]

  • 7개 데이터셋 체계적 비교, 5개 SOTA
  • KA + ETS 조합이 간결하면서 효과적
  • Domain encoder >> General LLM 실증

[Weaknesses]

  • TASTEset 실패 (-3.84, -5.83) 과소평가: "lexically distinct entities" 설명 불충분
  • ETS inference N배 비용 논의 없음 (ERFG 10 type = 10배)
  • Knowledge context가 수동 작성: 자동화 미탐구
  • 문장 단위만: cross-sentence 의존성 못 잡음 (레시피 이해에 치명적)
  • LLM 비교가 few-shot only: LoRA fine-tuned LLM과 비교 안 함
  • SINERA/SINERAS (Sony AI, 2024) 등 최근 baseline 누락

[Relevance]

  • LG Oven cooking action, tool, temperature, duration 추출 → 오븐 파라미터 매핑 기반
  • SoC RAG GPT-4o보다 BERT-base가 28점 높음 → on-device encoder model 정당화
  • KG 기초연구 entity type이 KG triple의 node/edge type에 직접 대응
  • K-Culture 연구 기회: cross-sentence recipe understanding, KA context 자동 생성, KG 통합

Paper 3 — ★★★

NG-Router: Graph-Supervised Multi-Agent Collaboration for Nutrition Question Answering

Kaiwen Shi, Zheyuan Zhang, Zhengqing Yuan, Keerthiram Murugesan, Chuxu Zhang, Yanfang Ye
EACL 2026 Long Notre Dame / IBM arXiv: 2510.09854 Nutrition KG
[Figure] See paper PDF — NG-Router architecture: Heterogeneous GNN + gradient-based subgraph retrieval

1. 전체 요약

Nutrition QA를 graph-supervised multi-agent routing 문제로 정의한다. 도메인 KG의 entity 노드와 24개 agent 노드(6 strategy × 4 LLM)를 하나의 heterogeneous graph로 통합하고, gradient-based subgraph retrieval로 70% 노드/79% 에지를 pruning하여 sparse 환경에서 F1 45.52 → 75.29 (+65%)의 대폭 향상을 달성한다.

2. 기존 연구 흐름

  • GraphRouter (Feng+, ICLR 2025): heterogeneous graph + GNN으로 LLM routing — NG-Router의 직접 전신
  • RouteLLM (Ong+, ICLR 2025), RouterDC (Chen+, NeurIPS 2024): LLM routing
  • Think-on-Graph (Sun+, ICLR 2024): KG 기반 LLM reasoning
  • NGQA Benchmark (Zhang+, ACL 2025): 같은 연구 그룹의 nutrition QA 벤치마크

3. 이 논문의 차별점

  • Domain KG + Agent graph 통합: 기존 LLM routing은 query feature만 사용, NG-Router는 nutrition entity를 graph에 포함
  • Gradient-based pruning: routing loss의 gradient norm으로 salient entity 식별 — 학습 가능한 graph pruning
  • Soft supervision: 24개 agent 전부 실행하여 empirical performance를 soft target distribution으로 변환

4. Method 상세

Heterogeneous Graph Construction

  • Query 노드 + Agent 노드 (6 strategy × 4 LLM = 24개) + Entity 노드 (food, user profile, health condition, nutrition tag)
  • Heterogeneous GNN (2-3 layers, hidden 256)으로 node embedding
  • MLP로 (query, agent) routing score → softmax → routing distribution

Gradient-Based Subgraph Retrieval

  • Entity embedding에 대한 KL loss gradient norm 계산
  • 높은 gradient = 중요 entity → threshold 0.01로 pruning
  • Sparse setting: 70% 노드, 79% 에지 제거, SNR 210% 향상

7. 결론

Nutrition QA에서 domain KG를 multi-agent routing graph에 통합하면 sparse 환경에서도 강력한 성능을 보인다. Gradient-based subgraph pruning은 noise 제거와 효율화를 동시에 달성한다.

8. 내 의견

[Strengths]

  • KG + multi-agent를 heterogeneous graph로 우아하게 통합
  • Sparse에서 +65% 향상은 인상적
  • Gradient-based pruning이 실용적

[Weaknesses]

  • Self-referential benchmark: NGQA가 같은 연구 그룹 산출물 — 외부 검증 없음
  • 24 agents × 전체 학습 데이터 inference = 331K+ LLM 호출 — 비용 논의 없음
  • Simple BM25 + GPT-4o-mini baseline 부재
  • US-centric dietary data only — 한국/아시아 전이 불명
  • Gradient-based pruning 자체는 GNNExplainer 등에서 기존 기법
  • "Reasoning"이라기보다 noise 제거에 가까움

[Relevance]

  • LG Oven nutrition KG + multi-agent routing 아키텍처 참고
  • SoC RAG subgraph pruning (70% 감소)은 on-device 효율화에 직접 적용 가능
  • KG 기초연구 NGQA를 한국 식이 데이터로 확장하면 후속 연구
  • Glocal Lab Simple RAG + single agent가 이 복잡한 시스템을 이기면 counterpoint 논문 가능

"Yuki Gets Sushi, David Gets Steak?": Uncovering Gender and Racial Biases in LLM-Based Meal Recommendations

Xuefeng Wei, Xuan Zhou, Yusuke Sakai, Taro Watanabe
EACL 2026 Long NAIST GitHub: Nutrition-Bias-Dataset Food AI Bias
[Figure] See paper PDF — Nutrition Bias Index visualization across racial groups

1. 전체 요약

LLM 기반 식사 추천 시스템의 성별/인종 편향을 최초로 체계적으로 정량 분석한다. 320개 이름 × 50 restart × 8 모델 = 128,000개 추천을 생성하여, Nutrition Bias Index/Score (NBI/NBS) 메트릭을 정의한다. 핵심 발견: Black-Hispanic 그룹이 통계적으로 구별 불가능한 고위험군으로 clustering되며, CoT reasoning이 편향을 악화시킨다 (명시적 스테레오타입 언어화).

2. 기존 연구 흐름

  • Nghiem+ (EMNLP 2024): 이름 기반 LLM 편향 프로빙 (320개 이름 세트의 원천)
  • Eloundou+ (ICLR 2025 Spotlight): chatbot 공정성
  • Zuo+ (2021): food-gender stereotype ("You are what you eat")
  • Luo+ (ICWSM 2024): 이민자 음식의 othering

3. 이 논문의 차별점

  • 최초의 식사 추천 편향 정량 분석: 기존 bias 연구는 직업/성격 중심, 식사 추천은 미개척
  • USChainMains 데이터셋: 78개 미국 체인 레스토랑, 780개 메뉴, 10개 영양 속성
  • NBI/NBS 메트릭: AHA 가이드라인 기반 편향 정량화
  • CoT가 편향 악화: reasoning이 명시적 stereotype 언어화로 이어짐

4. Method 상세

  • 실험 규모: 128,000 추천 (320 이름 × 50 restart × 8 모델)
  • 모델: DeepSeek-R1 (1.5B~671B), Gemini 2.0/2.5 Flash
  • 기준: USDA (≤600kcal), AHA (8개 영양소 threshold)
  • NBI: 각 영양소의 AHA 초과 비율을 그룹 간 비교
  • NBS: NBI의 가중 합계, 그룹별 overall bias score

GRPO 학습에서 balanced difficulty filtering이 학습 효율과 최종 성능을 동시에 개선한다. 이론적으로 variance-based learnability proxy가 이를 뒷받침하며, 실용적으로 training cost를 50% 이상 절감한다.

8. 내 의견

[Strengths]

  • 이론과 실험이 잘 정렬된 깔끔한 논문
  • +3.8 with <50% steps는 매우 실용적
  • Learnability = Var(p) 공식이 직관적이고 일반적

[Weaknesses]

  • 0.3-0.7 range의 최적성 증명이 empirical only — 이론적 bounds 부재
  • 3B에서만 검증 — 더 큰 모델에서의 효과 불명
  • Filtering으로 인한 데이터 utilization 감소의 장기 영향 미분석

[Relevance]

  • SoC RAG 경량 모델 RL training에 difficulty filtering 직접 적용 가능
  • KG 기초연구 KG reasoning task의 난이도 조절에 이 원리 적용 가능
